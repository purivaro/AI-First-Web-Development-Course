## ⚛️ ส่วนที่ 2 — รู้จัก Framework: React (และ Vue)
*(25 นาที)*

### 2.1 Framework คืออะไร? ทำไมต้องมี?
**Framework** = ชุดเครื่องมือ + วิธีการเขียนโค้ดที่จัดระเบียบไว้แล้ว ช่วยให้สร้างแอปใหญ่ๆ ได้ง่ายขึ้น

เปรียบเทียบ:
- เขียน JS เปล่าๆ = สร้างบ้านจากการเลื่อยไม้เอง
- ใช้ Framework = สร้างบ้านโดยมีชิ้นส่วนสำเร็จรูป + พิมพ์เขียวมาตรฐาน

**Framework ยอดนิยม:**
| Framework | จุดเด่น | ใครใช้ |
|---|---|---|
| **React** | ยืดหยุ่นสูง, ecosystem ใหญ่ที่สุด, AI ถนัดที่สุด | Facebook, Netflix, Airbnb |
| **Vue.js** | เรียนง่าย, syntax กระชับ | Alibaba, GitLab |
| **Svelte** | เร็ว, โค้ดน้อย | Apple (บางส่วน) |

### 2.2 ทำไมเราเน้น React?
1. **AI ถนัด React มากที่สุด** — ข้อมูลฝึก AI มี React เยอะกว่า framework อื่นเยอะ → AI สร้างโค้ด React ได้แม่นและรวดเร็วกว่า
2. **Community ใหญ่** — เจอปัญหาอะไร Google เจอคำตอบเร็ว
3. **เครื่องมือรอบข้างพร้อม** — Vite, Next.js, shadcn/ui, Tailwind ทำงานกับ React ได้ดี
4. **Google Stitch export โค้ดเป็น React + Tailwind** ได้ตรงๆ

> 📌 **ไม่ได้แปลว่า Vue/Svelte แย่** แค่ในยุค AI-First การเลือก stack ที่ AI ถนัดที่สุด = ทำงานเสร็จเร็วที่สุด

### 2.3 Component — หัวใจของ React
แนวคิดของ React: **"แบ่งหน้าเว็บเป็นชิ้นเล็กๆ ที่นำกลับมาใช้ใหม่ได้ (Component)"**

เปรียบเทียบกับ Lego:
- **Button Component** = ตัวต่อปุ่ม (ใช้ซ้ำได้ทุกที่)
- **Card Component** = ตัวต่อการ์ด
- **Page** = นำ Component หลายๆ ตัวมาประกอบกัน

ตัวอย่างโค้ด React (อ่านพอ ไม่ต้องเขียน):
```jsx
function TodoItem({ text }) {
  return <li className="p-2 bg-white rounded">{text}</li>;
}

function App() {
  return (
    <ul>
      <TodoItem text="ซื้อนม" />
      <TodoItem text="ออกกำลังกาย" />
    </ul>
  );
}
```

**สังเกต:**
- เขียนคล้าย HTML แต่อยู่ในไฟล์ `.jsx`
- ใช้ Tailwind class ได้ตรงๆ (`className="p-2 bg-white"`)
- Component ตัวเดียวเรียกใช้ซ้ำได้

### 2.4 วิธีสั่ง AI ให้ใช้ React (สำคัญที่สุด!)
**Pattern ที่ใช้ได้ผล:** ระบุ **Stack** ให้ชัดในคำสั่ง

❌ **แบบไม่ดี:**
> *"สร้างแอป To-do ให้หน่อย"*

✅ **แบบดี (Agent-style):**
> *"สร้างแอป To-do โดยใช้ **React + Vite + Tailwind CSS** เก็บข้อมูลด้วย **localStorage** ก่อน ขอให้:*
> - *แยกเป็น Component: `TodoItem`, `TodoList`, `AddTodoForm`*
> - *ใช้ React Hooks (`useState`, `useEffect`) สำหรับ state*
> - *Tailwind class สำหรับ styling*
> - *ก่อนเริ่มเขียน ขอให้ถามฉัน 3 คำถามถ้ามีจุดไม่ชัด"*

**คำศัพท์ที่ควรใส่ใน Prompt:**
- `React + Vite` — บอก framework + build tool
- `Tailwind CSS` — บอก styling system
- `useState` / `useEffect` — บอกวิธีจัดการ state แบบ React
- `Component` — บอกให้แบ่งโครงสร้างเป็นชิ้น

## 🧠 ส่วนที่ 3 — แนวคิด State: หัวใจของ Web App
*(20 นาที)*

### 3.1 State คืออะไร?
**State = ข้อมูลปัจจุบันที่หน้าเว็บ "จำได้" ขณะใช้งาน**

ตัวอย่าง State ในชีวิตจริง:
- จำนวนของในตะกร้า Shopee = 3 ชิ้น
- รายการ To-do ที่ยังไม่ได้ทำ = 5 รายการ
- ผู้ใช้กำลังล็อกอินอยู่ = `true`

**กฎทอง:** *State เปลี่ยน → UI เปลี่ยนตามอัตโนมัติ* (React จัดการให้)

### 3.2 ตัวอย่าง: Counter Component
```jsx
function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>คุณคลิกแล้ว {count} ครั้ง</p>
      <button onClick={() => setCount(count + 1)}>คลิกฉัน</button>
    </div>
  );
}
```

**อ่านโค้ดยังไง:**
- `useState(0)` = สร้าง state ชื่อ `count` เริ่มต้นที่ 0
- `setCount(count + 1)` = สั่งเปลี่ยนค่า count
- พอ state เปลี่ยน → React วาดหน้าเว็บใหม่ให้เอง

### 3.3 ข้อจำกัดของ State
- **State หายเมื่อปิดเว็บ/รีเฟรช** — เพราะอยู่ในหน่วยความจำชั่วคราว
- ถ้าอยากเก็บถาวร → ต้องมี **Database**

> 👉 นี่คือเหตุผลที่เราต้องเรียนเรื่อง Database ต่อ

## 🗄️ ส่วนที่ 5 — Database & CRUD
*(25 นาที)*

### 5.1 Database คืออะไร?
**Database = พื้นที่เก็บข้อมูล "ถาวร"** ที่อยู่บน Server ปิดเปิดเว็บกี่ทีข้อมูลก็ยังอยู่

เปรียบเทียบ:
- **State** = กระดาษโน้ตข้างโต๊ะ (หายเมื่อเก็บโต๊ะ)
- **Database** = แฟ้มเอกสารในตู้เซฟ (อยู่ถาวร)

### 5.2 ทางเลือก Database ที่ AI จัดการได้ง่าย
| ทางเลือก | จุดเด่น | เหมาะกับ |
|---|---|---|
| **Google Sheets** | ตั้งง่ายสุด, เห็นเป็นตาราง | โปรเจกต์ทดลอง, ข้อมูลน้อย |
| **Firebase Firestore** | Real-time, ฟรี tier ใจดี | แชท, To-do, แอปที่ต้อง sync |
| **Supabase** | PostgreSQL + UI สวย, ฟรี tier ดี | Web App ทั่วไป (แนะนำ) |
| **Cloudflare D1** | SQLite บน edge, เร็วมาก | Production scale |
| **localStorage** (ของ Browser) | ไม่ต้องตั้งอะไร แต่เก็บอยู่ในเครื่องผู้ใช้คนเดียว | Prototype, ทดลองก่อนต่อ DB จริง |

> 📌 **คาบนี้เริ่มจาก `localStorage` ก่อน** (ง่ายที่สุด ไม่ต้อง setup) แล้วค่อยอัปเกรดเป็น **Supabase** สำหรับโปรเจกต์จริง

### 5.3 CRUD — 4 การกระทำพื้นฐาน
ทุก Web App ที่จัดการข้อมูล ทำได้แค่ 4 อย่าง:
| ตัวอักษร | ภาษาไทย | ตัวอย่าง |
|---|---|---|
| **C**reate | สร้างใหม่ | เพิ่มรายการ To-do |
| **R**ead | อ่าน | ดูรายการ To-do ทั้งหมด |
| **U**pdate | แก้ไข | ติ๊กว่าทำเสร็จแล้ว |
| **D**elete | ลบ | ลบรายการที่ไม่ต้องการ |

> 🎯 **เคล็ดลับสั่ง AI:** บอก AI ตรงๆ ว่า *"ทำ CRUD ครบทั้ง 4 ตัวให้ด้วย"* — AI จะรู้ทันทีว่าต้องสร้างฟีเจอร์อะไรบ้าง
