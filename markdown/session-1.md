## 🎯 ส่วนที่ 1 — เปิดคาบ & แนวคิด "AI-First Web Architect"
*(20 นาที)*

### 1.1 แนวคิดนี้ต่างจากการเรียนเขียนโปรแกรมแบบเดิมอย่างไร?
ในอดีต หากคุณอยากสร้างเว็บ คุณต้องเริ่มต้นจากการท่องจำคำสั่ง (Syntax) เช่น การจำว่าพิมพ์ `<br>` คือการขึ้นบรรทัดใหม่ การสร้างหน้าเว็บต้องพิมพ์โค้ดทีละบรรทัด

**แต่ในยุค AI-First ทุกอย่างเปลี่ยนไปแล้ว:**
- **AI คือ "ผู้รับเหมาก่อสร้าง"**: AI สามารถเขียนโค้ดได้เก่งและเร็วกว่ามนุษย์ทั่วไปหลายสิบเท่า
- **คุณคือ "สถาปนิก (Architect)"**: หน้าที่ของคุณไม่ใช่การไปก่ออิฐทีละก้อน แต่คือ **การออกแบบ วางสเปก (Spec) เชื่อมต่อเครื่องมือ (Tools) และสั่งการ**
- **ทักษะที่สำคัญที่สุดเปลี่ยนไป**: จาก "การจำ Syntax โค้ด" กลายเป็น "การร่างความต้องการ (Requirements) ด้วย Markdown", "การเลือกใช้ระบบ Design (Stitch)", และ "การใช้ AI อัตโนมัติในงานซ้ำซาก (Git)"

### 1.2 เครื่องมือ (Tools) ที่เราจะใช้เป็น "ทีมงาน" ของเรา
1. **AI Assistant / Editor (เช่น Antigravity / Cursor / Claude)**: ทำหน้าที่เป็นมือขวาในการเขียนโค้ด
2. **Google Stitch**: ระบบเครื่องมือสร้าง Design Tokens และ UI ให้เรา
3. **MCP (Model Context Protocol)**: เทคโนโลยีตัวกลางที่ทำให้ AI สามารถ "เสียบปลั๊ก" ดึงข้อมูลจาก Google Stitch มาใช้เขียนโค้ดได้ตรงเป๊ะ
4. **Git & GitHub**: ระบบเซฟเกมและพื้นที่โชว์ผลงาน (Hosting) สำหรับมืออาชีพ

## 📝 ส่วนที่ 2 — Markdown ภาษากลางในการทำ "แบบแปลน" (Requirements)
*(25 นาที)*

### 2.1 ทำไมเรื่องนี้ถึงสำคัญมากในการสั่ง AI?
บริษัทเทคโนโลยีค้นพบว่า ปัญหาใหญ่ของการใช้ AI คือ **"อาการหลงลืมและหลงทาง (Context Drift)"** หากสั่งงานยาวๆ ในแชท AI จะเริ่มลืม

**🎯 ทางออก: เทคนิค Spec-Driven Development**
เราจะไม่สั่งให้ AI สร้างเว็บแบบลอยๆ แต่เราจะสร้างไฟล์ `.md` (เช่น `requirements.md`) เพื่อเป็น **Single Source of Truth (คัมภีร์หลัก)** ให้ AI อ่านเสมอ

### 2.2 ไวยากรณ์ (Syntax) เบื้องต้น
- ใช้ `# ` สำหรับหัวข้อหลัก
- ใช้ `- ` สำหรับรายการ (Bullet)
- ใช้ `**` คล่อมข้อความเพื่อทำตัวหนา

### 2.3 Workshop: ร่างแบบแปลนของฉัน
1. เปิด AI Editor ขึ้นมา
2. สร้างโฟลเดอร์โปรเจกต์ และสร้างไฟล์ `requirements.md`
3. เขียนรายละเอียดเว็บที่อยากได้ (เช่น หัวข้อ, โทนสี, เนื้อหาที่จะใส่) ลงในไฟล์นี้

## 🎨 ส่วนที่ 3 — ออกแบบด้วย Google Stitch & เชื่อมต่อ MCP
*(30 นาที)*

แทนที่จะมานั่งจินตนาการสีหรือระยะห่างเอง เราจะใช้เครื่องมือระดับโปรอย่าง Google Stitch!

### 3.1 สร้าง Design System ด้วย Stitch
- เข้าสู่ระบบ Google Stitch สร้างโปรเจกต์ใหม่ (เช่น "AI-First Web Architect")
- เลือกโทนสี (เช่น สีพื้นดำ, สีปุ่มฟ้านีออน) เลือกฟอนต์ (เช่น Space Grotesk)
- ระบบ Stitch จะสร้าง Design Tokens (ค่าสี, ฟอนต์, ระยะห่าง) ออกมาให้แบบสำเร็จรูป

### 3.2 เชื่อมต่อพลังด้วย MCP (Model Context Protocol)
นี่คือความลับของ Workflow นี้! แทนที่เราจะก๊อปปี้โค้ดสีมาเองทีละตัว เราจะสอนวิธีต่อสายตรง:
1. นำ API Key ของ Google Stitch มาตั้งค่าใน AI Editor ผ่าน MCP Configuration (เช่น ใน `mcp_config.json`)
2. เมื่อเชื่อมต่อเสร็จ AI Editor จะมีดวงตาที่สามารถ "มองเห็น" Design System ในบัญชี Stitch ของคุณได้โดยตรง!

## 🤖 ส่วนที่ 4 — ลงมือสร้าง: ให้ AI เนรมิตโค้ด
*(30 นาที)*

เมื่อเรามี **แบบแปลน (requirements.md)** และมี **ดีไซน์ (Stitch via MCP)** แล้ว ถึงเวลาสั่งผู้รับเหมาลงมือ!

### 4.1 คำสั่งเดียว (The Master Prompt)
เปิดแชทใน AI Editor แล้วพิมพ์คำสั่ง:
> *"ช่วยอ่านไฟล์ `requirements.md` เพื่อดูเนื้อหาของเว็บ จากนั้นให้ดึง Design System ชื่อ 'AI-First Web Architect' จาก Google Stitch ผ่าน MCP แล้วนำทั้งหมดมาเขียนเป็นโค้ดเว็บ (HTML/CSS/JS หรือ React/Vite) ให้สมบูรณ์แบบที่สุด"*

### 4.2 AI ทำงาน (Magic Moment)
AI จะประมวลผล:
1. อ่านเนื้อหาที่คุณร่างไว้
2. เชื่อมต่อไปดึงโทนสีและฟอนต์จาก Stitch
3. เขียนไฟล์โค้ด (เช่น `index.html` หรือ `App.jsx`) ขึ้นมาให้ทันที
- **ผลลัพธ์:** คุณจะได้เว็บที่สวยงามระดับโปรตรงตามที่ออกแบบไว้ โดยไม่ต้องพิมพ์โค้ดเอง!

## 🚀 ส่วนที่ 5 — Version Control & Deploy: เอาขึ้นออนไลน์แบบเหนือชั้น
*(30 นาที)*

ตอนนี้เว็บยังอยู่ในเครื่องเรา เราต้องส่งขึ้น "GitHub" เพื่อให้คนทั่วโลกเข้าถึงได้

### 5.1 เข้าใจ Git ใน 1 นาที
การทำงานกับ Git เหมือนการแพ็คของส่งไปรษณีย์:
- `git add .` = เอาของทั้งหมดใส่กล่อง
- `git commit` = ปิดกล่องและแปะป้ายชื่อ (บอกว่าอัปเดตอะไร)
- `git push` = ส่งกล่องขึ้นรถไปรษณีย์ (GitHub)

### 5.2 ให้ AI พิมพ์คำสั่ง Git ให้ (AI-Assisted Workflow)
แทนที่คุณจะต้องเปิดหน้าจอดำ (Terminal) มาพิมพ์คำสั่งเองทีละบรรทัด เราสามารถใช้พลังของ AI Automation ได้!
- **คำสั่งที่คุณพิมพ์บอก AI ในแชท:**
  > *"งานส่วนนี้เสร็จแล้ว ช่วยทำการ `git add .`, `git commit` ด้วยข้อความว่า 'สร้างหน้าเว็บแรกด้วย Stitch Design', และ `git push` โค้ดทั้งหมดขึ้น GitHub ให้ฉันที"*
- AI จะรันคำสั่งเหล่านี้เบื้องหลังให้อัตโนมัติ (อาจมี popup ถามยืนยันเพื่อความปลอดภัย) นี่คือประสบการณ์ของสถาปนิกตัวจริงที่ให้ลูกน้องจัดการงานน่าเบื่อ!

### 5.3 เปิดใช้งาน GitHub Pages
เมื่อโค้ดไปอยู่บน GitHub แล้ว ขั้นตอนสุดท้ายคือการเปิดสวิตช์ให้เป็นเว็บไซต์:
1. ไปที่หน้า GitHub Repository ของคุณ
2. ไปที่ **Settings > Pages**
3. หัวข้อ Build and deployment ตรง Branch ให้เลือก `main` (หรือ branch หลักของคุณ)
4. กด Save และรอประมาณ 1 นาที
5. รีเฟรชหน้าเว็บ คุณจะได้ URL ของจริง เช่น `https://ชื่อคุณ.github.io/my-website` เอาไปอวดเพื่อนได้เลย!
