# 🎨 คาบเรียนที่ 2: Mastering Web Layout by Working with AI

**ระยะเวลา:** 2 ชั่วโมง 45 นาที
**กลุ่มเป้าหมาย:** นิสิตไม่มีพื้นฐานการเขียนโปรแกรม
**แนวคิดหลัก:** "นักออกแบบไม่ใช่คนที่วาดรูปเก่ง แต่คือคนที่จัดวางองค์ประกอบเป็น" — ในคาบนี้เราจะไม่ให้นิสิตเขียน CSS เอง แต่จะสอนให้เข้าใจว่า **"ความสวยงามของเว็บเกิดจากอะไร"** แล้วใช้ AI อย่าง **Google Stitch** เป็นผู้ช่วยเนรมิตดีไซน์ออกมาเป็นโค้ด

---

## 🎯 เป้าหมายของคาบเรียน (Learning Outcomes)
1. นิสิตเข้าใจโครงสร้างพื้นฐานของ UI เว็บ (Box Model, Spacing, Typography, สี)
2. นิสิตสามารถใช้งาน **Google Stitch** เปลี่ยนไอเดียในหัวให้กลายเป็น UI Design ที่นำไปใช้ได้จริง
3. นิสิตสามารถสั่ง AI (ใน Antigravity) ให้อ่านและอธิบายโค้ด CSS ของดีไซน์นั้นๆ ได้
4. นิสิตสามารถปรับแต่ง Layout และทำให้เว็บรองรับการแสดงผลบนมือถือ (Responsive) ได้

## 📘 รายละเอียดเนื้อหาการสอน

### 1. ทบทวนและเตรียมความพร้อม (15 นาที)
- ชื่นชมผลงานจากคาบแรก: เปิด GitHub Pages ของนิสิตบางคนขึ้นโชว์
- ทบทวนหัวใจสำคัญ: "ไฟล์ `requirements.md` คือแผนที่ ถ้าไม่มีแผนที่ AI จะหลงทาง"

### 2. Layout 101: ภาษาของนักออกแบบเว็บ (30 นาที)
*เป้าหมาย: นิสิตไม่จำเป็นต้องเขียนโค้ดเป็น แต่ต้อง **"เรียกชื่อ"** ให้ถูก เพื่อสั่ง AI ได้แม่นยำ*
- **The Box Model (ทุกอย่างคือกล่อง):**
  - **Content:** เนื้อหาข้างใน (ตัวหนังสือ, รูปภาพ)
  - **Padding (เบาะกันกระแทก):** ระยะห่างจาก "เนื้อหา" ถึง "ขอบกล่อง" (ทำให้ดูไม่อึดอัด)
  - **Border:** เส้นขอบ
  - **Margin (ระยะเว้นวรรค):** ระยะห่างจาก "กล่องนี้" ถึง "กล่องอื่น"
- **Container & Section:** การแบ่งโซนของเว็บ (เช่น โซน Header, โซน Hero, โซน Footer)
- **Flexbox & Grid (ภาพรวม):** ระบบจัดแถวของกล่อง (เรียงซ้าย-ขวา หรือเรียงบน-ล่าง)
- **Spacing & Typography:** เว็บจะดู "แพง" หรือ "เชย" อยู่ที่การเว้นช่องไฟและขนาดฟอนต์

### 3. พระเอกของงาน: ใช้งาน Google Stitch (45 นาที)
*เป้าหมาย: ใช้ AI ระดับสูงในการเจนดีไซน์ โดยที่นิสิตสามารถควบคุมได้*
- **Google Stitch คืออะไร:** เครื่องมือออกแบบ UI ด้วย AI จาก Google ที่ต่างจาก ChatGPT ตรงที่มันมีหน้าจอให้เราคลิกปรับแต่งแบบ Visual ก่อนจะได้โค้ด (เสมือนมี Designer ส่วนตัว)
- **Step-by-Step:**
  1. เข้าเว็บ Google Stitch และล็อคอิน
  2. ใส่ Prompt: ให้นำไฟล์ `requirements.md` จากการบ้านคาบที่แล้ว ไปแปะในช่องคำสั่งของ Stitch
  3. **การเลือก Theme:** สอนนิสิตใช้เครื่องมือของ Stitch ปรับแต่ง Color Palette (Primary, Surface, Background) และเลือกฟอนต์
  4. **Micro-adjustments:** ใช้เมาส์คลิกที่ส่วนต่างๆ ของ UI บน Stitch แล้วสั่ง Prompt สั้นๆ เช่น *"เปลี่ยนรูปนี้ให้กลม", "เพิ่มพื้นที่ว่างตรงนี้ให้ดูมินิมอลขึ้น"*
  5. **Export to Code:** เมื่อพอใจกับหน้าตาแล้ว กดปุ่มให้ Stitch คายผลลัพธ์ออกมาเป็นโค้ด HTML/CSS (หรือ Tailwind)

### 4. Code & Refine: นำโค้ดมาปรับแต่งใน Editor (30 นาที)
*เป้าหมาย: รู้จักคุยกับโค้ดผ่าน Antigravity*
- นำโค้ดที่ได้จาก Google Stitch มาวางในโปรแกรม **Antigravity**
- **เทคนิคการใช้ AI อ่านโค้ด:**
  - ไฮไลต์โค้ด CSS บางส่วนที่น่าสงสัย (เช่น คลาสแปลกๆ)
  - สั่ง AI ใน Antigravity ว่า *"ช่วยอธิบายบรรทัดนี้หน่อยว่ามันทำหน้าที่อะไรบนหน้าเว็บ?"*
- **ทำ Responsive (รองรับมือถือ):**
  - สั่ง AI ว่า *"ช่วยปรับโค้ดชุดนี้ให้เปิดบนหน้าจอมือถือแล้วไม่พังที"*
  - ให้ AI อธิบายกลับมาว่ามันเติมโค้ด `flex-col` หรือ `@media` ตรงไหนบ้าง

### 5. Workshop: Redesign โฉมใหม่หมดจด (30 นาที)
*ภารกิจที่ต้องทำส่งในห้อง:*
1. นำ requirement ของหน้าเว็บแนะนำตัวเดิม ไปสร้าง UI ใหม่ใน **Google Stitch** ให้ดูมีความเป็นมืออาชีพขึ้น (Modern & Clean)
2. นำโค้ดที่ได้มาใส่ในโปรเจกต์เดิมใน Antigravity
3. **บังคับ:** ต้องสั่ง AI ใน Antigravity ให้ปรับแก้สีหรือช่องไฟในโค้ด **อย่างน้อย 1 จุด** โดยใช้คำศัพท์ที่เรียนไป (เช่น Margin, Padding)
4. อัปเดตงานขึ้น GitHub ด้วยคำสั่ง **Git CLI** (`git add .`, `git commit -m "redesign with stitch"`, `git push`)
5. ส่งลิงก์ผลงาน!

### 6. สรุปบทเรียน (15 นาที)
- **สรุป Concept:** การใช้ Google Stitch เป็นการทำ **"Rapid Prototyping"** หรือการขึ้นโครงแบบรวดเร็ว
- **ข้อควรระวัง:** AI มักจะใส่โค้ดรกๆ หรือเกินความจำเป็นมาให้ หน้าที่ของเราคือการโยนโค้ดเข้า Antigravity แล้วสั่งให้ AI อธิบาย หรือ refactor (จัดระเบียบ) ให้สะอาดขึ้น
- **เป้าหมายต่อไป:** คาบหน้าเราจะเรียนวิธีทำให้หน้าเว็บนิ่งๆ พวกนี้ "ขยับได้และตอบสนองได้" ด้วย JavaScript!
