### 2. CSS เบื้องต้น + Layout 101 + Tailwind (40 นาที)
*เป้าหมาย: นิสิตไม่จำเป็นต้องเขียนโค้ดเป็น แต่ต้อง **"เรียกชื่อ"** ให้ถูก เพื่อสั่ง AI ได้แม่นยำ และอ่านโค้ดที่ AI สร้างมาออก*

#### 2.1 CSS คืออะไร?
- **HTML = โครงกระดูก, CSS = ผิวหนัง/เสื้อผ้า** — CSS คือกฎที่บอกเบราว์เซอร์ว่า "element ไหนหน้าตาแบบไหน" (สี, ขนาด, ตำแหน่ง, ระยะห่าง)
- **โครงสร้างพื้นฐาน:**
  ```css
  selector {
    property: value;
  }
  ```
  ตัวอย่าง: `p { color: blue; font-size: 18px; }` = ย่อหน้าทุกอันเป็นสีน้ำเงิน ตัวอักษร 18px

- **3 วิธีใส่ CSS:**
  1. **Inline** บน tag: `<p style="color: red;">` — ใช้น้อยที่สุด
  2. **`<style>` block** ใน `<head>` — ใช้ตอนทดลอง
  3. **ไฟล์ `.css` แยก** เช่น `style.css` แล้ว link เข้าหน้า HTML — **Best practice**

#### 2.2 Selector แบบที่เจอบ่อย
| Selector | หมายถึง | ตัวอย่าง |
|---|---|---|
| `p`, `h1` | เลือกทุก tag ชนิดนั้น | `p { ... }` |
| `.ชื่อ-class` | เลือก element ที่มี class นั้น | `.btn-primary { ... }` |
| `#ชื่อ-id` | เลือก element ที่มี id นั้น (1 ตัวเดียว) | `#header { ... }` |

#### 2.3 The Box Model (ทุกอย่างคือกล่อง)
- **Content:** เนื้อหาข้างใน (ตัวหนังสือ, รูปภาพ)
- **Padding (เบาะกันกระแทก):** ระยะห่างจาก "เนื้อหา" ถึง "ขอบกล่อง" (ทำให้ดูไม่อึดอัด)
- **Border:** เส้นขอบ
- **Margin (ระยะเว้นวรรค):** ระยะห่างจาก "กล่องนี้" ถึง "กล่องอื่น"

> 💡 **เคล็ดลับ:** เปิด DevTools (คลิกขวา → Inspect) แล้วเลือก element ใดก็ได้ จะเห็นกล่อง Box Model แสดงเป็นภาพประกอบสีๆ ให้ดูชัดเจน

#### 2.4 Layout System — จัดวางหลายกล่องยังไง
- **Container & Section:** การแบ่งโซนของเว็บ (เช่น โซน Header, โซน Hero, โซน Footer)
- **Flexbox:** ระบบจัดแถวของกล่องในแกนเดียว (เรียงซ้าย-ขวา หรือบน-ล่าง) — เหมาะกับ Navbar, การ์ดเรียงข้างกัน
- **Grid:** ระบบจัดเป็นตาราง 2 แกน (แถว × คอลัมน์) — เหมาะกับ Dashboard, แกลเลอรีรูป
- **Spacing & Typography:** เว็บจะดู "แพง" หรือ "เชย" อยู่ที่การเว้นช่องไฟและขนาดฟอนต์

#### 2.5 รู้จัก Tailwind CSS — ทางลัดของยุค AI
**ปัญหาของ CSS แบบดั้งเดิม:** ต้องสลับไปเขียนในไฟล์ `.css` แยก, ต้องคิดชื่อ class เอง, โค้ดยาว และเมื่อโปรเจกต์ใหญ่ขึ้น CSS จะรกมาก

**Tailwind CSS คืออะไร?**
- **Utility-first CSS framework** — เขียน class สำเร็จรูปตรงๆ บน HTML แทนการเขียน CSS แยก
- ตัวอย่าง:
  ```html
  <!-- CSS แบบเดิม -->
  <div class="card">...</div>
  <style>
    .card { padding: 16px; background: blue; color: white; border-radius: 8px; }
  </style>

  <!-- Tailwind -->
  <div class="p-4 bg-blue-500 text-white rounded-lg">...</div>
  ```

**ทำไม Tailwind เหมาะกับยุค AI?**
1. **ไม่ต้องสลับไฟล์** — เห็นทั้งโครงสร้างและสไตล์ในที่เดียว
2. **AI สร้างเร็วและแม่นกว่า** — Tailwind มี class ตายตัว AI ไม่ต้องคิดชื่อใหม่
3. **Google Stitch export ออกมาเป็น Tailwind อยู่แล้ว** — เป็นมาตรฐานของวงการ
4. **ปรับแก้ทันใจ** — เปลี่ยน `p-4` → `p-8` รู้เลยว่า padding เพิ่ม ไม่ต้องตามหาในไฟล์ CSS

**Class Tailwind ที่เจอบ่อย (ไม่ต้องท่อง แค่อ่านออก):**
| กลุ่ม | ตัวอย่าง class | ความหมาย |
|---|---|---|
| Spacing | `p-4`, `m-2`, `px-6`, `py-3` | padding/margin (เลขใหญ่ = ระยะเยอะ) |
| Layout | `flex`, `grid`, `gap-4` | จัดวางกล่อง |
| Size | `w-full`, `h-screen`, `max-w-md` | ความกว้าง/สูง |
| Text | `text-xl`, `font-bold`, `text-center` | ขนาด/น้ำหนัก/จัดวางตัวอักษร |
| Color | `bg-blue-500`, `text-white` | สีพื้นหลัง / สีตัวอักษร |
| Effect | `rounded-lg`, `shadow-md`, `hover:bg-blue-700` | มุมโค้ง, เงา, เอฟเฟกต์ตอน hover |
| Responsive | `md:flex`, `lg:text-2xl` | ใช้ class นี้เฉพาะหน้าจอขนาด md/lg ขึ้นไป |

> 🎯 **Key Takeaway:** เราไม่ต้องเขียน Tailwind เป็น แต่ต้อง "อ่านออก" ว่า `class="p-4 bg-blue-500"` แปลว่าอะไร เพื่อสั่ง AI ปรับแต่งได้ถูกจุด

### 3. พระเอกของงาน: ใช้งาน Google Stitch + MCP (40 นาที)
*เป้าหมาย: ใช้ AI ระดับสูงในการเจนดีไซน์ โดยที่นิสิตสามารถควบคุมได้ และเชื่อม Stitch กับ AI Editor ด้วย MCP*

#### 3.1 Google Stitch คืออะไร
เครื่องมือออกแบบ UI ด้วย AI จาก Google ที่ต่างจาก ChatGPT ตรงที่มันมีหน้าจอให้เราคลิกปรับแต่งแบบ Visual ก่อนจะได้โค้ด (เสมือนมี Designer ส่วนตัว)

#### 3.2 Step-by-Step การใช้ Stitch
1. เข้าเว็บ Google Stitch และล็อคอิน
2. ใส่ Prompt: ให้นำไฟล์ `requirements.md` จากการบ้านคาบที่แล้ว ไปแปะในช่องคำสั่งของ Stitch
3. **การเลือก Theme:** สอนนิสิตใช้เครื่องมือของ Stitch ปรับแต่ง Color Palette (Primary, Surface, Background) และเลือกฟอนต์
4. **Micro-adjustments:** ใช้เมาส์คลิกที่ส่วนต่างๆ ของ UI บน Stitch แล้วสั่ง Prompt สั้นๆ เช่น *"เปลี่ยนรูปนี้ให้กลม", "เพิ่มพื้นที่ว่างตรงนี้ให้ดูมินิมอลขึ้น"*
5. **Export to Code:** เมื่อพอใจกับหน้าตาแล้ว กดปุ่มให้ Stitch คายผลลัพธ์ออกมาเป็นโค้ด HTML/CSS (หรือ Tailwind)

#### 3.3 เชื่อมต่อพลังด้วย MCP (Model Context Protocol)
การใช้ MCP คือจิ๊กซอว์ชิ้นสำคัญที่ทำให้ AI ฝั่ง Design (Stitch) และฝั่ง Code (Antigravity) สามารถ **"คุยกันเอง"** ได้โดยตรง — แทนที่จะต้อง Export โค้ดมาแปะทีละรอบ

**กลไกการทำงาน:**
1. **ฝั่ง Stitch ทำหน้าที่เป็น "MCP Server"** เปิดช่องทาง (Endpoint) ให้ AI ตัวอื่นเข้ามาอ่าน Design System ได้โดยตรง (Layout, ฟอนต์, Color Tokens, Variables)
2. **ฝั่ง Antigravity ทำหน้าที่เป็น "MCP Client"** มีเครื่องมือเชื่อมต่อไปยัง Stitch Server ทำให้ "มองเห็น" ข้อมูลล่าสุดในโปรเจกต์ออกแบบได้ตลอดเวลา
3. **Real-time Syncing:** เมื่อมีการปรับดีไซน์ (เช่น เปลี่ยนสีหลักจากส้มเป็นเหลือง) Agent ฝั่ง Antigravity ตรวจจับและเสนออัปเดต CSS / Design Tokens ในโค้ดให้ทันที

**ตัวอย่างคำสั่งที่กระชับขึ้นมาก:**
> *"ดึงดีไซน์หน้า Dashboard ล่าสุดจาก Stitch แล้วนำมาเขียนโค้ดอัปเดตให้ตรงกัน"*

Agent จะเรียก API ไปคุยกับ Stitch ทันทีเพื่อดึงโครงสร้างมาจัดการให้ — การสลับบริบทระหว่าง Design กับ Code แทบจะหายไปเลย

### 4. Code & Refine: นำโค้ดมาปรับแต่งใน Editor (30 นาที)
*เป้าหมาย: รู้จักคุยกับโค้ดผ่าน Antigravity*
- นำโค้ดที่ได้จาก Google Stitch มาวางในโปรแกรม **Antigravity**
- **เทคนิคการใช้ AI อ่านโค้ด:**
  - ไฮไลต์โค้ด CSS บางส่วนที่น่าสงสัย (เช่น คลาสแปลกๆ)
  - สั่ง AI ใน Antigravity ว่า *"ช่วยอธิบายบรรทัดนี้หน่อยว่ามันทำหน้าที่อะไรบนหน้าเว็บ?"*
- **ทำ Responsive (รองรับมือถือ):**
  - สั่ง AI ว่า *"ช่วยปรับโค้ดชุดนี้ให้เปิดบนหน้าจอมือถือแล้วไม่พังที"*
  - ให้ AI อธิบายกลับมาว่ามันเติมโค้ด `flex-col` หรือ `@media` ตรงไหนบ้าง

### 5. Workshop: Redesign โฉมใหม่หมดจด (30 นาที)
*ภารกิจที่ต้องทำส่งในห้อง:*
1. นำ requirement ของหน้าเว็บแนะนำตัวเดิม ไปสร้าง UI ใหม่ใน **Google Stitch** ให้ดูมีความเป็นมืออาชีพขึ้น (Modern & Clean)
2. นำโค้ดที่ได้มาใส่ในโปรเจกต์เดิมใน Antigravity
3. **บังคับ:** ต้องสั่ง AI ใน Antigravity ให้ปรับแก้สีหรือช่องไฟในโค้ด **อย่างน้อย 1 จุด** โดยใช้คำศัพท์ที่เรียนไป (เช่น Margin, Padding)
4. อัปเดตงานขึ้น GitHub ด้วยคำสั่ง **Git CLI** (`git add .`, `git commit -m "redesign with stitch"`, `git push`)
5. ส่งลิงก์ผลงาน!

### 6. สรุปบทเรียน (10 นาที)
- **สรุป Concept:** การใช้ Google Stitch เป็นการทำ **"Rapid Prototyping"** หรือการขึ้นโครงแบบรวดเร็ว
- **ข้อควรระวัง:** AI มักจะใส่โค้ดรกๆ หรือเกินความจำเป็นมาให้ หน้าที่ของเราคือการโยนโค้ดเข้า Antigravity แล้วสั่งให้ AI อธิบาย หรือ refactor (จัดระเบียบ) ให้สะอาดขึ้น
- **เป้าหมายต่อไป:** คาบหน้าเราจะเรียนวิธีทำให้หน้าเว็บนิ่งๆ พวกนี้ "ขยับได้และตอบสนองได้" ด้วย JavaScript!
