## 🎯 ส่วนที่ 1 — เปิดคาบ & แนวคิด "AI-First Web Development"
*(20 นาที)*

### 1.1 แนวคิดนี้ต่างจากการเรียนเขียนโปรแกรมแบบเดิมอย่างไร?
ในอดีต หากคุณอยากสร้างเว็บ คุณต้องเริ่มต้นจากการท่องจำคำสั่ง (Syntax) เช่น การจำว่าพิมพ์ `<br>` คือการขึ้นบรรทัดใหม่ หรือ `color: red;` คือการเปลี่ยนสีข้อความ กว่าจะได้เว็บหนึ่งหน้าต้องพิมพ์โค้ดหลายร้อยบรรทัดด้วยตัวเอง

**แต่ในยุค AI-First ทุกอย่างเปลี่ยนไปแล้ว:**
- **AI คือ "ช่างก่อสร้าง"**: AI รุ่นใหม่ๆ อย่าง Claude 4.6 Sonnet, Gemini 3.1 หรือ GPT-5.5 (หรือรุ่นที่ใหม่กว่า) สามารถเขียนโค้ดได้เก่งและเร็วกว่ามนุษย์ทั่วไปหลายสิบเท่า
- **คุณคือ "สถาปนิก" และ "ผู้กำกับ"**: หน้าที่ของคุณไม่ใช่การไปก่ออิฐทีละก้อน (เขียนโค้ดทีละบรรทัด) แต่หน้าที่ของคุณคือ **การออกแบบ ควบคุม ตรวจสอบ และสั่งการ** ให้ช่างก่อสร้าง (AI) ทำงานออกมาตรงตามที่คุณต้องการ
- **ทักษะที่สำคัญที่สุดเปลี่ยนไป**: จาก "การจำ Syntax โค้ด" กลายเป็น "การสื่อสารความต้องการให้ชัดเจน (Prompting)" และ "การเข้าใจโครงสร้างระบบภาพรวม"

### 1.2 เครื่องมือ (Tools) แถวหน้าของโลกที่เราจะใช้เป็น "ผู้ช่วย"
ในหลักสูตรนี้ เราไม่ได้ผูกติดกับเครื่องมือใดเครื่องมือหนึ่ง แต่เพื่อให้เห็นภาพความสามารถสูงสุด ณ ปัจจุบัน (ข้อมูลอัปเดตปี 2026) นี่คือ 3 ค่ายยักษ์ใหญ่ที่เราเลือกใช้งานได้:

1. **Claude (claude.ai) จากค่าย Anthropic** 
   - **รุ่นล่าสุด:** Claude 4.6 Sonnet (รุ่นยอดฮิต) / 4.7 Opus (รุ่นท็อป)
   - **ความสามารถเด่น:** ได้รับการยอมรับว่าเป็น "เบอร์หนึ่งด้านการเขียนโค้ด" เก่งเรื่องการทำงานที่ซับซ้อน (Agentic Workflows) รับข้อมูลบริบทได้มหาศาล และมีระบบ **Artifacts** ที่สามารถจำลองหน้าเว็บ (Render) โชว์ให้เราดูและกดเล่นได้ทันทีในหน้าแชท *(แนะนำให้ใช้ตัวนี้เป็นหลักในวิชานี้)*

2. **ChatGPT (chatgpt.com) จากค่าย OpenAI**
   - **รุ่นล่าสุด:** GPT-5.5 / GPT-5.5 Instant
   - **ความสามารถเด่น:** เป็น AI ที่เก่งรอบด้านที่สุด (All-rounder) มีความคิดเชิงตรรกะและการวิเคราะห์ที่ก้าวกระโดด (Complex Reasoning) อาการมั่วหรือหลอน (Hallucination) ต่ำลงมาก เหมาะสำหรับช่วยออกแบบสถาปัตยกรรมระบบ แก้บั๊กยากๆ และเขียนโค้ดที่ต้องการตรรกะระดับสูง

3. **Gemini จากค่าย Google**
   - **รุ่นล่าสุด:** ซีรีส์ Gemini 3.1 (เช่น Pro และ Flash-Lite)
   - **ความสามารถเด่น:** ตอบสนองได้รวดเร็วมาก (Ultra-low latency) และเก่งที่สุดในด้าน **Multimodal** (การประมวลผล ภาพ วิดีโอ เสียง พร้อมๆ กับโค้ด) แถมยังเชื่อมต่อดึงข้อมูลปัจจุบันจาก Google Search ได้แบบเรียลไทม์

## 🌐 ส่วนที่ 2 — พื้นฐานที่ต้องรู้: เว็บทำงานอย่างไร?
*(25 นาที)*

แม้เราจะมี AI คอยเขียนโค้ดให้ แต่ในฐานะ "ผู้กำกับ" เราต้องรู้ว่าโครงสร้างหลักของเว็บมีอะไรบ้าง เพื่อให้เราสั่งการและแก้ไขปัญหาได้ถูกจุด

### 2.1 เปรียบเทียบเว็บไซต์กับ "การสร้างบ้าน"
เว็บไซต์ทุกเว็บในโลก ไม่ว่าจะเป็น Google, Facebook หรือเว็บเล็กๆ ล้วนประกอบขึ้นจาก 3 ภาษาหลัก เปรียบได้กับการสร้างบ้านดังนี้:

**1. HTML (HyperText Markup Language) = โครงสร้างบ้าน 🧱**
- **หน้าที่:** กำหนดว่าหน้าเว็บมี "องค์ประกอบ" อะไรบ้าง
- **เปรียบเทียบ:** เหมือนเสา คาน หลังคา ประตู หน้าต่าง ของบ้าน
- **ตัวอย่างในเว็บ:** มีหัวข้อใหญ่, มีรูปภาพ, มีปุ่มกด, มีตาราง
- **หน้าตาโค้ด:** จะอยู่ในวงเล็บมุม เช่น `<h1>นี่คือหัวข้อ</h1>`, `<button>คลิกที่นี่</button>`

**2. CSS (Cascading Style Sheets) = การตกแต่งบ้าน 🎨**
- **หน้าที่:** กำหนด "หน้าตาและความสวยงาม" ให้กับองค์ประกอบ HTML
- **เปรียบเทียบ:** เหมือนการทาสีผนัง การเลือกสีผ้าม่าน การจัดวางเฟอร์นิเจอร์
- **ตัวอย่างในเว็บ:** ตัวอักษรสีน้ำเงิน, ปุ่มมีขอบโค้งมน, รูปภาพอยู่ตรงกลางหน้าจอ
- **หน้าตาโค้ด:** `color: blue;`, `font-size: 20px;`, `border-radius: 10px;`

**3. JavaScript (JS) = ระบบไฟฟ้าและประปา ⚡**
- **หน้าที่:** กำหนด "พฤติกรรมและการโต้ตอบ (Interactive)"
- **เปรียบเทียบ:** เหมือนสวิตช์ไฟที่พอกดแล้วหลอดไฟสว่าง หรือก๊อกน้ำที่เปิดแล้วน้ำไหล
- **ตัวอย่างในเว็บ:** คลิกปุ่มแล้วมีหน้าต่างเด้งขึ้นมา (Popup), การคำนวณราคาสินค้าเมื่อกดใส่ตะกร้า, สไลด์โชว์รูปภาพเลื่อนได้

> **💡 เคล็ดลับสำหรับ AI-First:** 
> คุณ **ไม่ต้องจำ** วิธีเขียนโค้ดเหล่านี้เลย! สิ่งที่คุณต้องรู้คือคำศัพท์ เช่น ถ้าหน้าเว็บสีไม่สวย คุณต้องบอก AI ว่า *"ช่วยแก้ **CSS** เปลี่ยนสีปุ่มให้หน่อย"* หรือถ้าปุ่มกดแล้วเงียบ คุณต้องบอก AI ว่า *"ช่วยเช็ค **JavaScript** ของปุ่มนี้ที มันกดไม่ติด"*

### 2.2 วงจรการทำงานของเว็บไซต์
1. คุณพิมพ์ URL (เช่น `www.example.com`) ใน Browser (Chrome, Safari)
2. Browser วิ่งไปหา **Server** (เครื่องคอมพิวเตอร์ที่เก็บไฟล์เว็บไว้ตลอด 24 ชม.)
3. Server จะส่งไฟล์ **HTML, CSS, JS** ของเว็บนั้นกลับมา
4. Browser ของคุณทำหน้าที่ "ประกอบร่าง" ไฟล์โค้ดเหล่านั้น ให้กลายเป็นหน้าเว็บสวยๆ ที่คุณมองเห็น

## 📝 ส่วนที่ 3 — Markdown ภาษากลางในการคุยกับ AI
*(30 นาที)*

### 3.1 Markdown คืออะไร?
เวลาเราพิมพ์ข้อความปกติใน Notepad ตัวอักษรจะเรียบๆ เท่ากันหมด ไม่มีหัวข้อใหญ่ ไม่มีตัวหนา
**Markdown** คือ "วิธีการเขียนข้อความธรรมดา ให้มีรูปแบบ (Format) ที่สวยงามและเป็นระเบียบ" โดยการเติม **"สัญลักษณ์พิเศษ"** เข้าไปเล็กน้อย

### 3.2 ทำไมเรื่องนี้ถึงสำคัญมากในการสั่ง AI?
AI ถูกฝึก (Train) มาให้อ่านและเขียนข้อมูลในรูปแบบ Markdown เป็นหลัก
- **ถ้าคุณพิมพ์ข้อความติดๆ กันไปหมด:** AI จะแยกไม่ออกว่าตรงไหนคือหัวข้อ ตรงไหนคือเงื่อนไขสำคัญ ทำให้ AI สร้างงานออกมาผิดพลาดได้ง่าย
- **ถ้าคุณเขียนเป็น Markdown:** AI จะเข้าใจ "โครงสร้างความคิด" ของคุณทันที ว่านี่คือหัวข้อหลัก นี่คือรายการย่อย นี่คือข้อจำกัด ส่งผลให้ได้คำตอบที่แม่นยำขึ้นมหาศาล

> สังเกตไหมว่าเวลา AI ตอบคำถามเรา มันมักจะมีตัวหนา ตัวเอียง มีหัวข้อชัดเจน... นั่นแหละครับ AI กำลังตอบเราด้วย Markdown!

### 3.3 ไวยากรณ์ (Syntax) ที่ใช้บ่อยสุดๆ (ต้องจำแค่นี้!)

| ผลลัพธ์ที่ต้องการ | พิมพ์สัญลักษณ์นี้ | ตัวอย่าง | ผลที่ได้ตอนแสดงผล |
|---|---|---|---|
| **หัวข้อหลัก (ใหญ่สุด)** | ใส่ `# ` ข้างหน้า | `# หน้าแรกของเว็บ` | <h1>หน้าแรกของเว็บ</h1> |
| **หัวข้อรอง** | ใส่ `## ` ข้างหน้า | `## ส่วนแนะนำตัว` | <h2>ส่วนแนะนำตัว</h2> |
| **ตัวหนา (เน้นคำ)** | ใส่ `**` คล่อมคำ | `ปุ่มสี **แดง**` | ปุ่มสี **แดง** |
| **รายการ (Bullet)** | ใส่ `- ` หรือ `* ` | `- มีรูปโปรไฟล์`<br>`- มีชื่อใต้รูป` | <ul><li>มีรูปโปรไฟล์</li><li>มีชื่อใต้รูป</li></ul> |
| **รายการแบบตัวเลข** | ใส่ `1. `, `2. ` | `1. เข้าสู่ระบบ`<br>`2. กดปุ่ม` | <ol><li>เข้าสู่ระบบ</li><li>กดปุ่ม</li></ol> |

### 3.4 เปรียบเทียบการเขียน Prompt ให้ AI สร้างเว็บ

❌ **Prompt แบบคนทั่วไป (แบบเรียงความ):**
> "ช่วยทำหน้าเว็บพอร์ตฟอลิโอให้หน่อย เอาชื่อนายสมชาย รักดี เป็นนักศึกษาวิดวะ มีรูปภาพอยู่ตรงกลาง แล้วก็มีส่วนที่เป็นผลงาน 3 อย่าง แล้วข้างล่างมีปุ่มให้กดติดต่อสีฟ้าๆ สวยๆ หน่อยนะ"

✅ **Prompt แบบ AI-First (ใช้ Markdown จัดโครงสร้าง):**
> ช่วยสร้างโค้ดหน้าเว็บ Portfolio (ไฟล์เดียว HTML+CSS) โดยมีรายละเอียดดังนี้:
> 
> `# โครงสร้างของเว็บ`
> `## 1. Hero Section (ส่วนบนสุด)`
> `- **ข้อความหลัก:** "นายสมชาย รักดี"`
> `- **ข้อความรอง:** "นักศึกษาวิศวกรรมศาสตร์ ผู้หลงใหลในเทคโนโลยี"`
> `- **รูปภาพ:** ใส่รูป placeholder วงกลมไว้ตรงกลาง`
> 
> `## 2. Portfolio Section (ส่วนผลงาน)`
> `- แสดงกล่องผลงาน **3 กล่อง** เรียงต่อกันแนวนอน (ถ้าจอเล็กให้เรียงแนวตั้ง)`
> 
> `## 3. Footer (ส่วนล่างสุด)`
> `- มีปุ่มเขียนว่า "ติดต่อฉัน"`
> `- **ดีไซน์:** ปุ่มสีฟ้า (#007BFF) ขอบมน เวลานำเมาส์ไปชี้ให้เปลี่ยนเป็นสีเข้มขึ้น`

*อ่านแล้วเห็นความต่างไหมครับ? แบบหลัง AI จะทำตามได้เป๊ะมากแบบแทบไม่ต้องแก้เลย!*

## 🤖 ส่วนที่ 4 — ลงมือสร้าง Landing Page และศิลปะการสั่งแก้ (Iterative Prompting)
*(30 นาที)*

ในส่วนนี้เราจะนำความรู้เรื่อง Markdown มาใช้จริง เพื่อให้ AI สร้างเว็บไซต์ให้เราภายในไม่กี่วินาที

### 4.1 วิวัฒนาการปี 2026: จาก Prompt Engineering สู่ "Context Engineering"
ในปี 2026 โลกของการพัฒนาซอฟต์แวร์ด้วย AI ก้าวข้ามยุคของการนั่งประดิษฐ์คำสั่ง (Prompt Engineering) ไปสู่ยุคที่เรียกว่า **Context Engineering (วิศวกรรมการจัดการบริบท)** อย่างเต็มรูปแบบ

บริษัทเทคโนโลยีชั้นนำค้นพบว่า ปัญหาใหญ่ที่สุดของการใช้ AI เขียนโปรแกรม คือ **"อาการหลงลืมและหลงทาง (Context Drift & Hallucination)"** หากคุณสั่งงาน AI ยาวๆ ในช่องแชท AI จะเริ่มลืมเป้าหมายแรกๆ และตัวคุณเองก็จะลืมด้วยว่าเคยตกลงฟีเจอร์อะไรไว้บ้าง

**🎯 ทางออกของปี 2026: เทคนิค Spec-Driven Development ด้วยไฟล์ Markdown**
นักพัฒนาในปัจจุบันเปลี่ยนวิธีทำงานใหม่หมด โดยใช้ไฟล์ `.md` (เช่น `requirements.md` หรือ `spec.md`) เป็น **Single Source of Truth (แหล่งอ้างอิงความจริงเดียว)**

**หลักคิด 3 ข้อในการใช้ .md คุม AI ให้ได้ผลลัพธ์ระดับโปร:**
1. **Treat Context as Code (ให้ความสำคัญกับ .md เท่ากับโค้ด):** ไฟล์ `.md` คือ "สมองส่วนความจำระยะยาว" ของ AI ถ้าเขียนไฟล์ `.md` ได้มีโครงสร้างชัดเจน (มี `#` หัวข้อหลัก, `-` ลิสต์) AI จะอ่านและประมวลผลได้แม่นยำแบบไม่มีวันพลาด
2. **Plan-First Protocol (บังคับวางแผนก่อนเขียนโค้ด):** ⚠️ *ห้ามสั่งให้ AI เขียนโค้ดทันที!* ต้องคุยในแชทให้ได้ข้อสรุป แล้วสั่งให้ AI นำข้อสรุปนั้นไป **"อัปเดตลงไฟล์ `.md` ก่อนเสมอ"** เพื่อให้สถาปนิก (ตัวคุณ) มีโอกาสตรวจแปลนบ้านก่อนลงมือก่อสร้าง
3. **Negative Space Documentation (กำหนดข้อห้ามให้ชัด):** ในไฟล์ `.md` ยุคใหม่ ไม่ได้ระบุแค่สิ่งที่ "ต้องทำ" แต่นิยมเพิ่มหัวข้อ `## สิ่งที่ห้ามทำ (Constraints)` เช่น "ห้ามใช้ JavaScript ซับซ้อน", "ห้ามใช้สีแดง" การตีกรอบให้ AI จะช่วยลดความผิดพลาดได้มหาศาล

**🗣️ ตัวอย่างการพูดคุยที่ถูกต้อง (The Golden Workflow ของปี 2026):**
- ❌ **แบบอดีต (เสี่ยงพัง):** *"ช่วยแก้โค้ดเพิ่มปุ่มให้หน่อย"* (AI เข้าไปแก้โค้ดเลย ถ้าเข้าใจผิดคือพังเลย และไม่มีบันทึกว่าสั่งแก้ตอนไหน)
- ✅ **แบบปัจจุบัน (ปลอดภัย 100%):** 
  1. **[💬 แชท = ห้องประชุม]** *"อยากเพิ่มปุ่มบนเว็บ ควรวางตรงไหนและใช้สีอะไรดี?"* (ปรึกษาหาทางออกในแชทก่อน)
  2. **[📄 .md = รายงานการประชุม]** *"โอเค เอาตามที่คุณแนะนำ ช่วยอัปเดตไฟล์ `requirements.md` เพิ่มรายละเอียดและข้อจำกัดเรื่องปุ่มนี้เข้าไปให้ที"* (บังคับ AI บันทึกแบบแปลน)
  3. **[💻 โค้ด = หน้างานก่อสร้าง]** *"ไฟล์ requirement อัปเดตเรียบร้อยละ เช็คแล้วถูกต้อง งั้นตอนนี้ช่วยอัปเดตไฟล์ `index.html` ให้ตรงตาม requirement ล่าสุดได้เลย"* (AI เขียนโค้ดตามแบบแปลน 100%)

### 4.2 การเตรียม "วัตถุดิบ" ด้วย AI (Content & Image Generation)
ข้อผิดพลาดของมือใหม่คือการสั่ง AI ให้สร้างเว็บทั้งๆ ที่ยังไม่มีเนื้อหาอยู่ในหัว! ในวิชานี้เราจะใช้ AI เป็นคนคิดเนื้อหาให้เราก่อน

**Step-by-Step เตรียมเนื้อหาและรูปภาพ:**
1. **ให้ AI เป็น Copywriter:** เปิด ChatGPT, Gemini หรือ Claude บน Browser
2. **พิมพ์ Prompt บังคับโครงสร้าง Markdown:**
   *"ฉันชื่อ... อยากทำเว็บ Portfolio ตัวเอง ช่วยเขียนประวัติแนะนำตัวสั้นๆ พร้อมลิสต์ความสนใจ 3 ข้อ **ข้อสำคัญ: ให้ตอบกลับมาเป็นรูปแบบ Markdown โดยใช้ `#` เป็นชื่อหัวข้อใหญ่ และ `-` เป็นลิสต์รายการ เพื่อให้ฉันเอาไปทำเว็บต่อได้ง่ายๆ**"*
   *(การสั่งแบบนี้เป็นการฝึกให้นิสิตเห็นความเชื่อมโยงว่า เมื่อ AI พ่น Markdown ออกมา เราสามารถก๊อปปี้ไปใช้เขียนเว็บต่อได้ทันที)*
3. **ให้ AI เป็น Artist:** สั่งในแชทเดิม หรือเปิดเว็บวาดรูป (เช่น Copilot) ว่า *"ช่วยวาดรูปอวาตาร์ 3D มินิมอล โทนสีฟ้าพาสเทล สำหรับใช้ทำรูปโปรไฟล์หน่อย"* แล้วเซฟรูปลงเครื่อง

### 4.3 เริ่มประกอบร่างคัมภีร์ Requirement
เมื่อมีวัตถุดิบ (เนื้อหา + รูป) แล้ว เราจะนำมาประกอบเป็นแบบแปลน:
**Step 1:** เปิดโปรแกรม **Antigravity จาก Google** (Editor หลักของเรา)
**Step 2:** พิมพ์สั่งในช่องแชทว่า: *"ช่วยสร้างโฟลเดอร์ชื่อ `my-website` บน Desktop และสร้างไฟล์ชื่อ `requirements.md` ไว้ข้างในให้หน่อย"*
**Step 3:** สร้างโฟลเดอร์ย่อยชื่อ `images/` แล้วนำรูปอวาตาร์ที่ได้ไปใส่ไว้
**Step 4:** เปิดไฟล์ `requirements.md` ขึ้นมา นำ "เนื้อหาแบบ Markdown" ที่ AI แต่งให้จากข้อ 4.2 มาวาง และเพิ่มอธิบายสไตล์ดีไซน์ลงไป (เช่น โทนสี, รูปแบบ) แล้วกดเซฟ
*(ตอนนี้คุณมี Blueprint ที่สมบูรณ์พร้อมสั่งก่อสร้างแล้ว!)*

### 4.4 ให้ Antigravity ลงมือสร้างเว็บจาก Requirement
เมื่อเรามีแบบแปลนที่ชัดเจนและเป็นลายลักษณ์อักษร AI จะทำงานได้แม่นยำมาก

**Step-by-Step การสั่ง AI สร้างโค้ด:**
**Step 1:** ในโปรแกรม Antigravity เปิดช่องแชทขึ้นมา
**Step 2:** พิมพ์คำสั่งสั้นๆ ว่า: *"ช่วยอ่านไฟล์ `requirements.md` แล้วสร้างไฟล์ `index.html` ขึ้นมา โดยเขียนโค้ดให้ตรงตามความต้องการทั้งหมดที"*
**Step 3:** Antigravity จะทำการวิเคราะห์ไฟล์ `.md` อย่างละเอียด และลงมือเขียนไฟล์ `index.html` ให้จนเสร็จแบบอัตโนมัติ
**Step 4:** ไปที่ Desktop > โฟลเดอร์ `my-website` > ดับเบิ้ลคลิกไฟล์ `index.html` เพื่อเปิดดูผลลัพธ์บน Browser

### 4.5 ฝึกซ้อมการอัปเดตงาน (Iterative Cycle)
เรามาฝึกกระบวนการ **"คุย -> จด -> โค้ด"** ที่อธิบายไว้ในหัวข้อ 4.1 กันครับ

**Step-by-Step การเพิ่มฟีเจอร์ใหม่:**
**Step 1 (คุย):** พิมพ์ในแชทว่า *"ฉันอยากเพิ่มส่วนแกลอรี่รูปภาพ 4 รูปใต้โปรไฟล์ ช่วยเอาไอเดียนี้ไปอัปเดตลงในไฟล์ `requirements.md` ให้หน่อย"*
**Step 2 (จด):** Antigravity จะเข้าไปแก้ไฟล์ `.md` ให้ (เพิ่มหัวข้อแกลอรี่) ให้คุณเปิดไฟล์เช็คดูว่า AI บันทึกเป้าหมายถูกต้องไหม
**Step 3 (โค้ด):** เมื่อ Requirement ถูกต้อง ค่อยพิมพ์สั่งว่า *"เยี่ยมมาก ช่วยอัปเดตไฟล์ `index.html` ให้มีแกลอรี่ตาม requirement ล่าสุดที"*
**Step 4:** สลับไปที่ Browser แล้วกด Refresh (F5) เว็บจะเปลี่ยนไปทันที โดยที่ข้อมูลการทำแกลอรี่จะถูกบันทึกไว้อย่างปลอดภัยในไฟล์ `.md` ตลอดไป!

## 🚀 ส่วนที่ 5 — เอาขึ้นออนไลน์ด้วย GitHub Pages (Deploy)
*(30 นาที)*

ตอนนี้เว็บของคุณอยู่แค่ "ในเครื่องคอมพิวเตอร์ของคุณ" (เปิดดูได้แค่คนเดียว) เราต้องนำมันไปไว้บนอินเทอร์เน็ต กระบวนการนี้เรียกว่า **"การ Deploy"**

ในวิชานี้เราจะใช้ **GitHub** ซึ่งเป็นเว็บไซต์ที่โปรแกรมเมอร์ทั่วโลกใช้เก็บโค้ด (เปรียบเสมือน Google Drive ของนักพัฒนา) ข้อดีของการใช้ GitHub คือ **"มันอัปเดตเวอร์ชันใหม่ได้ง่ายมาก"** แถมยังเก็บประวัติการแก้โค้ดไว้ให้เราด้วย

### 5.1 สมัครสมาชิกและสร้าง Repository (พื้นที่เก็บโค้ด)
1. เปิด Browser ไปที่ **[github.com](https://github.com)** แล้วสมัครสมาชิกให้เรียบร้อย
2. เมื่อล็อคอินเข้ามาแล้ว ให้มองหาปุ่ม **"New"** สีเขียว (ด้านซ้ายบน) เพื่อสร้าง Repository ใหม่
3. **Repository name:** ตั้งชื่อโปรเจกต์ เช่น `my-first-website` (ห้ามเว้นวรรค ให้ใช้ขีดกลางแทน)
4. เลือกระดับความปลอดภัยเป็น **Public** (สาธารณะ) เพื่อให้เปิดเป็นเว็บได้
5. เลื่อนลงมาล่างสุด กดปุ่ม **Create repository** สีเขียว
6. คุณจะเห็นหน้าจอที่มีคำสั่งโค้ดมากมาย (ให้เปิดหน้านี้ค้างไว้ก่อน)

### 5.2 อัปโหลดโค้ดขึ้น GitHub ด้วย Git Commands (วิถีโปรแกรมเมอร์)
ในโลกการทำงานจริง เราจะไม่ใช้วิธีลากวางไฟล์บนเว็บ แต่เราจะใช้การพิมพ์คำสั่ง (Command Line) เพื่อส่งไฟล์จากในเครื่องเราขึ้นไปบน GitHub โดยตรง

**Step-by-Step การส่งไฟล์:**
1. เปิด Terminal (หน้าจอดำๆ) ในโปรแกรม **Antigravity**
2. พิมพ์คำสั่ง 3 ทหารเสือ เพื่อนำไฟล์ทั้งหมดเตรียมส่งขึ้นคลังข้อมูล:
   - `git add .` *(หมายถึง: นำไฟล์ทั้งหมดที่แก้ล่าสุด ใส่ตะกร้าเตรียมไว้)*
   - `git commit -m "first commit"` *(หมายถึง: ปิดกล่องพัสดุ แล้วแปะป้ายกำกับว่านี่คืองานครั้งแรก)*
3. กลับไปที่หน้าเว็บ GitHub (จากข้อ 5.1) ให้ก๊อปปี้คำสั่ง 3 บรรทัดที่ GitHub เตรียมไว้ให้ ไปวางใน Terminal แล้วกด Enter:
   - `git branch -M main`
   - `git remote add origin https://github.com/ชื่อคุณ/my-first-website.git`
   - `git push -u origin main` *(คำสั่งนี้คือการดันกล่องพัสดุขึ้นไปเก็บบนเว็บ GitHub)*
4. กลับไปหน้าเว็บ GitHub แล้วกด Refresh หน้าจอ (F5) คุณจะเห็นไฟล์โค้ดของคุณขึ้นไปอยู่บนเว็บแล้ว!

### 5.3 เสกให้กลายเป็นเว็บไซต์จริง (เปิดใช้งาน GitHub Pages)
ตอนนี้โค้ดอยู่บนอินเทอร์เน็ตแล้ว แต่ยังเป็นแค่ "ไฟล์ข้อความ" เราต้องสั่งให้ GitHub แปลงร่างมันเป็นเว็บไซต์:
1. ในหน้า Repository ของคุณ ให้คลิกที่แท็บ **Settings** (รูปฟันเฟือง ขวาบน)
2. มองเมนูด้านซ้ายมือ เลื่อนลงมาคลิกคำว่า **Pages**
3. ตรงส่วนที่เขียนว่า *Build and deployment* ใต้คำว่า *Branch* ให้คลิกปุ่มที่เขียนว่า `None` แล้วเปลี่ยนเป็น `main`
4. กดปุ่ม **Save**
5. รอประมาณ 1-2 นาที แล้วกด Refresh หน้าจอ (F5)
6. คุณจะเห็นข้อความแจ้งเตือนปรากฏขึ้นมาด้านบนว่า **"Your site is live at..."** พร้อมลิงก์เว็บไซต์ของคุณ!
*(ลิงก์จะหน้าตาเป็นมืออาชีพ เช่น `https://ชื่อคุณ.github.io/my-first-website/`)*

### 5.4 วิธีอัปเดตเว็บให้เป็นเวอร์ชันใหม่ (จุดเด่นของ Git)
นี่คือเหตุผลที่เราสอนใช้คำสั่ง Git! หากพรุ่งนี้คุณสั่ง AI ให้เพิ่มฟีเจอร์ใหม่ แล้วอยากอัปเดตเว็บให้คนอื่นเห็น มันจะง่ายกว่าเดิมมาก:
1. เปิด Terminal ใน Antigravity
2. พิมพ์คำสั่งเซฟงาน: `git add .` แล้วตามด้วย `git commit -m "อัปเดตเพิ่มหน้าแกลอรี่"`
3. พิมพ์คำสั่งดันงานขึ้นเว็บ: `git push`
4. รอ 1 นาที เว็บไซต์บนลิงก์เดิมของคุณจะเปลี่ยนเป็นเวอร์ชันใหม่ทันทีโดยไม่ต้องไปตั้งค่าอะไรเพิ่มอีกเลย!

## 🎬 ส่วนที่ 6 — Demo (โชว์ของ) สั่ง Deploy อัตโนมัติแบบ Pro + สรุป
*(15 นาที)*

### 6.1 Demo: โชว์ศักยภาพเมื่อใช้ AI จัดการ Git ให้แทน
ในหัวข้อ 5.2 คุณได้เรียนรู้วิธีพิมพ์คำสั่ง `git add`, `commit`, `push` ด้วยตัวเองไปแล้ว ซึ่งเป็นพื้นฐานที่สำคัญมาก
แต่ในโลกยุค AI-First เมื่อเราทำงานสเกลใหญ่ขึ้น เราสามารถให้ AI จัดการพิมพ์คำสั่งที่น่าเบื่อเหล่านี้ให้แทนได้!

*ผู้สอนเปิดจอโชว์ให้นิสิตดู:* 
1. ผู้สอนแก้ไขโค้ดเพิ่มฟีเจอร์ใหม่ผ่านโปรแกรม Antigravity
2. แทนที่จะพิมพ์คำสั่ง Git ใน Terminal เอง ผู้สอนแค่พิมพ์สั่ง AI ในช่องแชทของ Editor ว่า *"งานเสร็จแล้ว ช่วย Commit ด้วยข้อความว่า 'เพิ่มปุ่มล็อกอิน' แล้ว Push ขึ้น GitHub ให้หน่อย"*
3. AI จะรันคำสั่งเบื้องหลังให้เอง โค้ดวิ่งขึ้น GitHub ทันที!
4. ยิ่งไปกว่านั้น ถ้าเราเชื่อม GitHub เข้ากับระบบอย่าง Cloudflare Pages หรือ Vercel (ที่จะเรียนในคาบหลังๆ) เว็บไซต์จะทำการ Deploy ตัวเองเป็นเวอร์ชันใหม่โดยอัตโนมัติภายในเสี้ยววินาที

**ทำไมถึงโชว์ให้ดู?**
เพื่อให้ผู้เรียนเห็นว่า **เมื่อเราเข้าใจพื้นฐานคำสั่ง (Under the hood) แล้ว AI สามารถเป็น "ผู้ช่วย Automation" ที่ทำสิ่งซ้ำซากแทนเราได้** นี่คือพลังของการเป็นผู้กำกับ AI

### 6.2 สรุปสิ่งที่ได้เรียนรู้ในคาบที่ 1
- **Mindset:** เปลี่ยนจากการเป็น "คนเขียนโค้ด" ไปเป็น "ผู้กำกับ AI" 
- **Structure:** เว็บประกอบด้วย HTML (โครงกระดูก), CSS (ผิวพรรณ), และ JS (กล้ามเนื้อ/เส้นประสาท)
- **Context Engineering:** ต้องใช้ Markdown (`#`, `**`, `-`) บันทึก Requirement ก่อนเสมอ ป้องกัน AI อาการหลอน
- **Deployment:** การนำเว็บขึ้นออนไลน์และการเก็บประวัติโค้ดด้วย **GitHub** ทำให้อัปเดตงานง่ายและเป็นมืออาชีพ
